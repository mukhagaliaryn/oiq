# OIQ — Жоба жоспары

---

## Статустар
- `[ ]` — жасалу керек
- `[!]` — кезекті / қазір жасалуда
- `[x]` — дайын
- `[~]` — жартылай (блокер бар немесе кейінге қалдырылды)
- `[-]` — алынып тасталды

---

## Жалпы анықтамалар

**1-анықтама.** Платформада оқытушының 2 түрі бар. Қарапайым оқытушы (Regular) және 
жүйелік/серіктес оқытушы (System). Қарапайым оқытушы платформадағы өзіне берілген 
қолданушы мүмкіндіктерді пайдалана алады. Ал жүйелік оқытушы қолданушыдан бөлек деректер
қорына материалдар енгізу бойынша мүмкіндіктері бар. Жүйеге teacher типінде тіркелу 
кезінде бастапқы regular болып тіркеледі.

---

## Жалпы ескертулер

1. UI бойынша кез келген CRUD кезінде animate messages (top, left) шығу керек. Егер
өзгерту қажет болса, қазіргі кезде дайыны бар.
2. Белгілі бір мәтін енгізу кезінде бастапқы тілді en етіп gettext_lazy('...'). Ал
шаблондарда {% translate '...' %} арқылы жазу қажет.
3. Дизайн стиль бойынша: 
   - Платформа UI-жа Tailwind CSS, Alpine JS, HTMX, Lucide icons және т.б. қолданылған.    
   - Қажетті түстер мен мәндер айнымалылар түрінде styles.css бетінде көрсетілген. 
   - Осыған дейін жасалып қойған құрылымдық дизайн жүйесі бар: home, login, register және
   т.б. UI беттерінде. Соларға қарап келесі дизайн жүйесін құрылымдай беруге болады.
4. Платформаны әр түрлі құрылғыларға бейімді етіп жасау;
5. UI бойынша базалық html select ыңғайсыз. Сол себепті select-ті өзіміз қолмен құрастырып
шығуымыз керек. Дайын кастомды компонент: `ui/templates/components/select.html` (Alpine.js dropdown).
Пайдалану: `{% include "components/select.html" with field=form.field_name %}`;

### TASK-02: Teacher dashboard accounts
**Статус:** [x]
**Модуль:** apps.dashboard.teacher

**Не жасалу керек:**

Бұл тапсырмада оқытушының аккаунт бөлімін жасау керек керек. Аккаунт беттері 
teacher dashboard ішінде болады. Енді нақты қандай беттері мен функционалдықтар 
болатынын толығырақ айтып кетейін:

1. /profile/\<username>/. Қолданушының профил беті
2. /account/edit/. Қолданушының деректерін өзгеру беті. avatar-ға UI choice жасау керек.
3. /account/settings/ - Баптаулар беті. Тілді өзгерту және т.б.
4. /account/security/ - Қауіпсіздік беті. Бұл бетте парольді өзгерту, аккаунтты жою
және т.б.

**Ескертпелер:**

- Интерфейс бойынша креативтерді өзің басқада ресурстардан қарап, жалпы аккаунт беттер
UI бойынша қалай болатынын көріп жасай берсең болады.

### TASK-03: System teacher мүмкіндіктері
**Статус:** [x]
**Модуль:** apps.dashboard.teacher

**Не жасалу керек:**

Бұл тапсырмада жүйелік оқытушының қосымша мүмкіндіктерін жасайтын боламыз және бұл
қарапайым оқытушыға қатысты емес, оған доступ жоқ. Нақтырақ айтқанда:
- Оқытушы таңдаған пәні бойынша бөлімдер, тақырыптар және сұрақтар сала алады. Жалпы
таңдаған пәні бойынша мәселе: регистрация кезінде Teacher модельінде subject өрісі қалып
кеткен екен, оны қосу керек. Сонымен қатар teacher register формасына және басқа да 
қосу керек жерлерге қосып шығу;
- Әзірге superuser админ панельден оқытушыға system статусын қолмен береді. Сосын оған
осы мүмкіндіктер ашылады. Ол мүмкіндіктерді sidebar-ға тиісті 
шарттармен қою;
- Superuser тиісті статусты бергеннен соң dashboard-тағы sidebar-да оқытушыға тиісті пәніне 
сілтеме (subject link) ашылсын. Арқы қарай subject detail беті;
- Subject detail бетінде пән туралы анықтамалар және сол пәннің сыныптары (grades) tabs
түрінде шығу керек. Мысалы /subject/1?grade=5-class. Ол пәннің бөлімдерінің қай 
сыныптікі екендігін анықтап отыратын фильтер есебінде болады. Яғни: Математика пәні 
жалпы. Ол 5-7 сыныптарда болады. Оған тиісті жалпы бөлімдері бар. Сонымен қатар grade 
бойынша жеке сынып бөлімдері бар. subject -> grades -> chapters (grade) арқылы 
бөлімдерді бөліп аламыз. Сонда Математика. 5-сынып оқулығы есебінде бөлімдерге бөлінген, 
жек сынып пәні секілді әсер береді;
- Әр табтағы пән сыныптарының бөлімдерін according стилінде шығарамыз. Сонымен қатар
бөлімді өзгерту/өшіру әрекеттері және астыннан қоса алатындай етіп (alpine) жасап қою керек;
- Әр бөлім according-те тиісті тақырыптар тізімі болады. Ол тақырыптарды да қосу, өзгерту
және өшіре алатындай етіп жасау қажет;
- Chapter және Topic модельдерінде description өрісі бар. Соны қосу/өзгерту кезінде 
біздің жүйеге дайын oiq-ckeditor-5 rich text editor бар. Соны қолдану қажет;
- CRUD әрекеттерін барынша reload-сыз alpine немесе HTMX арқылы жасауды ұсынамын. Себебі
барлық әрекет бір бетте болғандықтан reload-ты көбейтпеген дұрыс

### TASK-04: Сұрақ қосу қосымшасы
**Статус:** [x]
**Модуль:** apps.dashboard.teacher
**Рөлі:** Жүйелік оқытушы

**Не жасалу керек:**

Бұл тапсырмада жүйелік оқытушы платформаға тиісті пән,бөлім,тақырып бойынша сұрақтар
енгізу беттерін жасау қажет. Сұрақ енгізу беттері нақты қандай болатыны жайлы айтып
кетейік:
- Subject detail бетінен Сұрақтар деген dropdown button болсын. Ол жерде Сұрақтар және
Сұрақ қосу деген 2 сілтеме болады. Сұрақтар пәнге тиісті сұрақтар тізімі беті болса, 
Сұрақ қосу сұрақ қосатын форма беті болады;
- **Сұрақтар беті**. Бұл бетте бірінші фильтер мен іздеу блогы және төменде сұрақтар тізімі
болады. Фильтерде сынып, бөлім, тақырып, format және variant бойынша фильтер болады. Сонымен 
қатар Question моделіне author деген өріс қосып қою керек. Себебі оқытушы тек өзі салған 
сұрақтар (default) сол пәннің тізімі бойынша және жоғарыда келтірілген фильтерлер бойынша 
шығып тұруы керек. Ал іздеу сұрақ берілгені text бойынша іздеу. Тізім бір бетте 100 сұрақтан 
тұратын пагинациялық тізім болсын. Әр item-де сұраққа сілтеме, өзгерту, өшіру батырмалары 
болады. Өшіру әрекеті modal арқылы жасау керек, ал өзгерту батырмасы сұрақ бетіне кетеді. 
Маңызды: осы жерде create және edit беттері жеке болады ма? әлде ортақ етіп ұйымдастыру керек 
пе? Соның қайсысы тиімді екенін шешіп, жасау керек;
- **Сұрақ беті** Сұрақ қосу бетінде 2 tab болсын. 
  - Бірінші tab (Сұрақ): сұрақтарды енгізу, форматы, формат түрі, text және т.б;
  - Екінші tab (Көрсетілім): Preview сұрақтың қалай шыққандығын көре алу және дұрыс/бұрыстығын
  тексеріп көру;
  - Сұрақтың форматына қарай (тест, сәйкестендіру, ...) қажетті жауап модельдері шығады.
  Қазірге тек тест форматы бар, сәйкесінше Option шығады. Сұрақты өзгерту (edit) кезінде 
  формат өзгермеуі керек;
  - Сақтау батырмасын басқанда Сұрақтар бетіне өту керек.
